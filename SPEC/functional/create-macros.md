# РЕЖИМ СОЗДАНИЯ MACROS

---

## ПОЛЬЗОВАТЕЛЬСКИЙ ПУТЬ

1. Запустил расширение
2. Включил режим создания сценария
   - Выбрал "По координатам" или "По элементам"
3. Сделал клики на странице
4. Выключил режим создания сценария
5. Опционально задал дополнительные параметры данного macros

---

## ЛОГИКА

### Запуск режима
- По кнопке в popup
- Предлагает пользователю варианты "По координатам" или "По элементам"
- После выбора варианта, считаем режим включённым, сворачивает popup и запоминает клики

### При режиме "по координатам"
- Расширение запоминает координаты кликов
- Не запоминает больше ничего
- По окончанию открывается окно редактирования macros, шаги = координаты

### При режиме "По элементам"
- Расширение запоминает selector элементов для клика
- Не запоминает больше ничего
- По окончанию открывается окно редактирования macros, шаги = selectors

### Слушатели событий
- `click` listener включается только во время записи
- Вне записи `recording-click` не отправляется
- Вне записи selector не строится
- После завершения записи listener снимается
- `keydown/keyup` не пишут клики
- Слушатели не блокируют события сайта
- Не использовать `stopPropagation`
- Не использовать `preventDefault`

### Завершение режима создания
- Когда пользователь завершил свои клики, то нажимает иконку расширения ещё раз, это является признаком завершения режима
- При этом открывается сразу popup с предзаполненными шагами "Окно редактирования macros"
- Название по умолчанию = `domain + date + time`, например `google.com 2026-06-02 19:34`. Без http, www, /...
- Пользователь может управлять этим окном как при редактировании уже существующими Macros
- Macros является созданным в этот момент. Закрытие окна не удаляет его, а лишь закрывает "как есть". Пользователь может удалить если захочет
